## Implementation Notes

### ESP Setup

**Recommended providers:** Loops or Kit (ConvertKit). Both handle:
- Automated sequences triggered on signup
- Conditional timing (compress for late subscribers)
- Open/click tracking
- Unsubscribe management (legally required)

### Supabase Integration

Current landing page inserts into `subscribers` table. Two options:

1. **Webhook on insert:** Supabase Edge Function triggers on new row, sends subscriber data to ESP via API
2. **Dual submit:** Landing page form submits to both Supabase and ESP simultaneously

Option 1 is cleaner (single source of truth). Option 2 is faster to implement.

### Tracking

- UTM parameter `?ref=` is already captured in the `source` column
- Add UTM tags to every email link: `?ref=email_1`, `?ref=email_5`, etc.
- Track which email drove the most landing page revisits and purchases

### Timing Rules

| Subscriber Joins | Sequence |
|---|---|
| 5+ weeks before launch | Full sequence (all 10 emails) |
| 3-4 weeks before launch | Skip Email 5, compress Phase 2 |
| 2 weeks before launch | Emails 1-4 (compressed to 4 days), then straight to countdown |
| 1 week before launch | Emails 1, 3, then countdown only |
| Launch week | Email 1 + Email 9 (launch day) + Email 10 |
